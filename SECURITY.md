# 安全性政策

## 回報漏洞

我們非常重視專案的安全性。若你發現任何安全性漏洞,**請勿開立公開 issue 或在公開討論區揭露**,以免在修復前被惡意利用。

請改用 GitHub 的私密漏洞回報功能私下回報:

- 開啟本倉庫的 **Security** 分頁,點選 **Report a vulnerability**(GitHub Security Advisories / GitHub private vulnerability reporting),即可私下提交漏洞報告。
- 非敏感、可公開討論的一般性錯誤,請改用一般的 GitHub issues 回報。

回報時若能附上以下資訊,將有助於我們更快重現與修復:

- 問題描述與潛在影響
- 重現步驟
- 受影響的版本或環境
- 任何相關的日誌、截圖或概念驗證

我們會儘快確認收到回報,並在評估與修復後協調公開揭露的時機。

## 支援版本

| 版本 | 是否支援 |
|---|---|
| 最新版本 | 是 |

> 安全性修復僅針對最新版本提供。建議使用者持續更新至最新版本。

## 安全考量

- 本程式需**系統管理員權限**執行(視窗擷取與輸入模擬),並會以系統層級**模擬全域滑鼠 / 鍵盤**。請僅在你自己的電腦、為個人用途執行;若執行期間焦點被切換到其他視窗,模擬的點擊 / 按鍵可能落在非預期的視窗。
- 本工具**完全離線**:不連線、不外送任何畫面、OCR 文字或設定到外部伺服器。
- `diagnostics/` 內的 elevated test runner 是**開發診斷工具,並非安全邊界**:它以管理員權限常駐、監看一個資料夾的 request 檔來觸發自動化。**請勿將本專案放在「Authenticated Users 可寫」的位置**(例如某些外接碟或磁碟根目錄的寬鬆 ACL),否則同機的低權限程序可能投遞 request、或竄改被執行的腳本 / `.venv` 解譯器而提權。建議將專案置於僅你本人與管理員可寫的位置(如使用者家目錄),或對專案資料夾收緊 ACL(例如 `icacls <專案目錄> /inheritance:r` 後只授予本人)。
